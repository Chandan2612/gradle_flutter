Gradle DSL — Groovy vs Kotlin

// Flutter 3.19+ defaults to Kotlin DSL (.kts files)

Kotlin DSL (`.kts`) is the modern standard. It provides type safety, IDE autocomplete, and better refactoring support. Groovy DSL (`.gradle`) is legacy but still widely used in older Flutter projects and plugins.

Root build.gradle (Kotlin DSL)

KOTLIN DSL — build.gradle.kts (root) 
// android/build.gradle.kts
plugins {
    // Apply false = declare but don't apply to root
    id("com.android.application") version "8.3.0" apply false
    id("com.android.library") version "8.3.0" apply false
    id("org.jetbrains.kotlin.android") version "1.9.22" apply false
}

allprojects {
    repositories {
        google()
        mavenCentral()
    }
}

App build.gradle (Kotlin DSL)

KOTLIN DSL — android/app/build.gradle.kts 
plugins {
    id("com.android.application")
    id("org.jetbrains.kotlin.android")
    id("dev.flutter.flutter-gradle-plugin")
}

android {
    namespace = "com.example.my_flutter_app"
    compileSdk = flutter.compileSdkVersion
    ndkVersion = flutter.ndkVersion

    defaultConfig {
        applicationId = "com.example.my_flutter_app"
        minSdk = flutter.minSdkVersion
        targetSdk = flutter.targetSdkVersion
        versionCode = flutter.versionCode
        versionName = flutter.versionName
        multiDexEnabled = true
    }

    buildTypes {
        release {
            isMinifyEnabled = true
            isShrinkResources = true
            proguardFiles(
                getDefaultProguardFile("proguard-android-optimize.txt"),
                "proguard-rules.pro"
            )
            signingConfig = signingConfigs.getByName("release")
        }
        debug {
            applicationIdSuffix = ".debug"
        }
    }

    compileOptions {
        sourceCompatibility = JavaVersion.VERSION_17
        targetCompatibility = JavaVersion.VERSION_17
    }

    kotlinOptions {
        jvmTarget = "17"
    }
}

flutter {
    source = "../.."
}

dependencies {
    implementation("androidx.multidex:multidex:2.0.1")
}

Dependencies & Repositories

// Know which keyword to use and where to get packages

Dependency configurations explained:
implementation — available only in this module (most common)
api — exposed to dependent modules (use sparingly)
compileOnly — compile-time only, not bundled
runtimeOnly — not available at compile time
testImplementation — only for unit tests
androidTestImplementation — only for instrumented tests

Common Flutter Dependencies

KOTLIN DSL — dependencies block 
dependencies {
    // MultiDex (required if minSdk < 21)
    implementation("androidx.multidex:multidex:2.0.1")

    // Firebase BOM (manages all Firebase versions)
    implementation(platform("com.google.firebase:firebase-bom:32.7.0"))
    implementation("com.google.firebase:firebase-analytics")
    implementation("com.google.firebase:firebase-crashlytics")

    // Local .aar file
    implementation(files("libs/my_sdk.aar"))
    // All .aar files in folder
    implementation(fileTree(mapOf("dir" to "libs", "include" to listOf("*.aar", "*.jar"))))

    // Test dependencies
    testImplementation("junit:junit:4.13.2")
    androidTestImplementation("androidx.test.espresso:espresso-core:3.5.1")
}

Version Catalog (libs.versions.toml)

TOML — gradle/libs.versions.toml 
# Declare all versions in one place
[versions]
agp = "8.3.0"
kotlin = "1.9.22"
multidex = "2.0.1"
firebase-bom = "32.7.0"

[libraries]
multidex = { group = "androidx.multidex", name = "multidex", version.ref = "multidex" }
firebase-bom = { group = "com.google.firebase", name = "firebase-bom", version.ref = "firebase-bom" }

[plugins]
android-application = { id = "com.android.application", version.ref = "agp" }
kotlin-android = { id = "org.jetbrains.kotlin.android", version.ref = "kotlin" }

# Usage in build.gradle.kts:
# dependencies { implementation(libs.multidex) }

Repositories Configuration

KOTLIN DSL — repositories block 
repositories {
    google()            // Android artifacts, Firebase
    mavenCentral()      // Most open-source libraries
    gradlePluginPortal() // Gradle plugins only

    // Private Maven repo (e.g. JFrog, Nexus)
    maven {
        url = uri("https://your-company.jfrog.io/artifactory/libs-release")
        credentials {
            username = providers.gradleProperty("NEXUS_USER").get()
            password = providers.gradleProperty("NEXUS_PASS").get()
        }
    }

    // JitPack for GitHub repos
    maven { url = uri("https://jitpack.io") }
}

Build Config, Flavors & Signing

// Release builds, product flavors, signing configs

Product Flavors

KOTLIN DSL — productFlavors 
android {
    flavorDimensions += listOf("environment")

    productFlavors {
        create("dev") {
            dimension = "environment"
            applicationIdSuffix = ".dev"
            versionNameSuffix = "-dev"
            buildConfigField("String", "BASE_URL", '"https://dev-api.example.com"')
        }
        create("staging") {
            dimension = "environment"
            applicationIdSuffix = ".staging"
            buildConfigField("String", "BASE_URL", '"https://staging-api.example.com"')
        }
        create("prod") {
            dimension = "environment"
            buildConfigField("String", "BASE_URL", '"https://api.example.com"')
        }
    }
    // Generates tasks: assembleProdRelease, assembleDevDebug, etc.
}

Signing Config (secure approach)

⚠️ Never hardcode keystore passwords in build.gradle. Use environment variables or a local keystore.properties file (gitignored).
KOTLIN DSL — signingConfigs 
// Read keystore.properties (gitignored)
val keystorePropsFile = rootProject.file("keystore.properties")
val keystoreProps = Properties()
if (keystorePropsFile.exists()) {
    keystoreProps.load(keystorePropsFile.inputStream())
}

android {
    signingConfigs {
        create("release") {
            storeFile = file(keystoreProps["storeFile"] as String)
            storePassword = keystoreProps["storePassword"] as String
            keyAlias = keystoreProps["keyAlias"] as String
            keyPassword = keystoreProps["keyPassword"] as String
        }
    }

    buildTypes {
        release {
            signingConfig = signingConfigs.getByName("release")
            isMinifyEnabled = true
            isShrinkResources = true
        }
    }
}

keystore.properties

PROPERTIES — (root)/keystore.properties [.gitignore this!] 
storeFile=../keystore/release.jks
storePassword=your_store_password
keyAlias=release_key
keyPassword=your_key_password

Errors & How to Fix Them

// 25+ real-world Gradle errors with step-by-step solutions

E
Namespace not specified
▼
ERROR MESSAGE
Namespace not specified. Specify a namespace in the module's build file.
// FIX — add namespace to android {} block
KOTLIN DSL
android {
    namespace = "com.example.myapp"  // ← add this
    compileSdk = 34
    ...
}
In older Groovy DSL: namespace 'com.example.myapp'. Required since AGP 7.3+. Also remove package attribute from AndroidManifest.xml if present.
E
minSdkVersion too low for dependency
▼
ERROR MESSAGE
uses-sdk:minSdkVersion X cannot be smaller than version Y declared in library
// FIX — increase minSdk in defaultConfig
KOTLIN DSL
defaultConfig {
    minSdk = 21  // ← bump to meet library requirement
    // OR override in manifest (not recommended):
    // tools:overrideLibrary="com.example.library"
}
E
Cannot resolve symbol / dependency not found
▼
ERROR MESSAGE
Could not resolve com.example:library:1.0.0
No cached version available for offline mode
// FIX — check repo, clear cache, force refresh
BASH
# Force refresh dependencies
./gradlew build --refresh-dependencies

# Clear Gradle cache
rm -rf ~/.gradle/caches/
# On Windows: del /s /q %USERPROFILE%\.gradle\caches\

# Also ensure repository is declared:
# repositories { google(); mavenCentral() }
E
Multidex — too many methods (> 64K)
▼
ERROR MESSAGE
Cannot fit requested classes in a single dex file.
method count: 72384 > 65536
// FIX — enable multiDex
KOTLIN DSL
defaultConfig {
    multiDexEnabled = true
}
dependencies {
    // Only if minSdk < 21
    implementation("androidx.multidex:multidex:2.0.1")
}
If minSdk ≥ 21, multidex is automatic — just set multiDexEnabled = true, no library needed.
E
AGP / Gradle version incompatibility
▼
ERROR MESSAGE
Android Gradle plugin requires Java 17 to run.
Minimum supported Gradle version is 8.0. Current version is 7.6.
// FIX — use the AGP compatibility matrix
COMPATIBILITY TABLE
AGP 8.3  → Gradle 8.4+  → Java 17+
AGP 8.2  → Gradle 8.2+  → Java 17+
AGP 8.0  → Gradle 8.0+  → Java 17+
AGP 7.4  → Gradle 7.5+  → Java 11+

# gradle-wrapper.properties:
distributionUrl=https\://services.gradle.org/distributions/gradle-8.4-all.zip

# settings.gradle.kts / root build.gradle.kts:
id("com.android.application") version "8.3.0"
E
NDK version mismatch
▼
ERROR MESSAGE
No version of NDK matched the requested version 21.4.7075529.
Requested by: project ':app'
// FIX — pin NDK version or install required version
KOTLIN DSL
android {
    // Pin to installed NDK version
    ndkVersion = "25.1.8937393"
    // OR use Flutter's recommended version:
    ndkVersion = flutter.ndkVersion
}
Install the NDK via: Android Studio → SDK Manager → SDK Tools → NDK. Or: sdkmanager "ndk;25.1.8937393"
E
Duplicate class error
▼
ERROR MESSAGE
Duplicate class kotlin.collections.jdk8.CollectionsJDK8Kt found in modules
kotlin-stdlib-jdk8 and kotlin-stdlib
// FIX — exclude or force resolve
KOTLIN DSL
dependencies {
    // Force consistent Kotlin stdlib version
    constraints {
        implementation("org.jetbrains.kotlin:kotlin-stdlib-jdk7:1.9.22") {
            because("kotlin-stdlib-jdk7 is now part of kotlin-stdlib")
        }
        implementation("org.jetbrains.kotlin:kotlin-stdlib-jdk8:1.9.22") {
            because("kotlin-stdlib-jdk8 is now part of kotlin-stdlib")
        }
    }
}
E
Jetifier / AndroidX conflict
▼
ERROR MESSAGE
Cannot access class 'android.support.v4.app.Fragment'.
Check your module classpath for missing or conflicting dependencies.
// FIX — enable Jetifier in gradle.properties
PROPERTIES
# gradle.properties
android.useAndroidX=true
android.enableJetifier=true

# Then run:
# flutter pub run jetifier:migrate
E
Out of Memory / Gradle Heap
▼
ERROR MESSAGE
java.lang.OutOfMemoryError: GC overhead limit exceeded
Java heap space
// FIX — increase JVM heap in gradle.properties
PROPERTIES
# gradle.properties
org.gradle.jvmargs=-Xmx4g -Xms512m -XX:MaxMetaspaceSize=512m -XX:+HeapDumpOnOutOfMemoryError
org.gradle.daemon=true
org.gradle.parallel=true
org.gradle.caching=true
E
Flutter plugin doesn't compile (plugin Groovy vs Kotlin)
▼
ERROR MESSAGE
Could not find method android() for arguments [plugin_build_gradle]
Script compilation errors in plugin build.gradle.kts
// FIX — check plugin build.gradle for required fields
GROOVY — minimal plugin build.gradle
group 'com.example.plugin'
version '1.0'

buildscript {
    repositories { google(); mavenCentral() }
    dependencies {
        classpath "com.android.tools.build:gradle:8.3.0"
    }
}

apply plugin: 'com.android.library'

android {
    namespace 'com.example.plugin'
    compileSdkVersion 34
    defaultConfig { minSdkVersion 16 }
}

1. Which file should you NEVER commit to Git in an Android/Flutter project?
build.gradle.kts
settings.gradle.kts
local.properties
gradle-wrapper.properties
✓ Correct! local.properties contains absolute SDK paths specific to each machine. Always gitignore it.

2. You want a dependency only available to modules that depend on yours (transitive). Which keyword do you use?
implementation
api
compileOnly
runtimeOnly
✓ Correct! api exposes the dependency transitively. Use sparingly — it increases build time.
✗ The answer is api. It exposes the dependency to consumers of your module.

Increase targetSdk
Enable ProGuard
Enable multiDexEnabled + add multidex library

✗ You need multiDexEnabled=true and the androidx.multidex:multidex library for old devices.

Gradle 8.0
Gradle 8.2
Gradle 8.4+
