Learning Roadmap

Learn Gradle in the right order.

Do not start by memorizing errors. First understand the build system layers. Then every error will tell you which layer is broken.

01

Flutter project root

Dart code, pubspec.yaml, assets, and platform folders live here.

02

Android folder

This is a real Android project inside your Flutter project.

03

Gradle wrapper

Locks the Gradle version used by this project.

04

AGP

Android Gradle Plugin adds Android build tasks to Gradle.

05

Kotlin plugin

Compiles Kotlin code used by Android plugins and native code.

06

Repositories

Gradle downloads plugins and libraries from configured repositories.

07

Dependencies

Libraries used by your app, Android plugins, tests, or build tools.

08

Debugging

Build Flow

What happens when you run flutter build apk?

Flutter handles Dart and framework work, then calls Android Gradle to produce APK or AAB.

1

Flutter command

You run flutter run, flutter build apk, or flutter build appbundle.

→
2

Gradle wrapper

Flutter calls android/gradlew, so your project Gradle version is used.

→
3

Settings file

settings.gradle loads plugin repositories and includes the app module.

→
4

→
5

Output

Version Compatibility

Gradle issues are mostly version mismatch issues.

Flutter, Gradle wrapper, AGP, Kotlin plugin, JDK, compileSdk, and dependencies must fit together. Do not blindly paste random versions from the internet.

Safe rule: use the versions generated by your current flutter create template first. Then upgrade one layer at a time.
Layer Where you see it Common problem Better fix
Gradle wrapper gradle-wrapper.properties AGP needs a newer Gradle version. Update wrapper based on AGP official compatibility.
AGP settings.gradle plugins block Old AGP cannot handle new SDK or namespace rules. Upgrade with Android Studio Upgrade Assistant or fresh Flutter template comparison.
Kotlin settings.gradle or build.gradle Binary metadata version mismatch. Update Kotlin plugin, but keep it compatible with AGP and Gradle.
JDK Android Studio Gradle JDK / flutter doctor -v Unsupported class file major version. Use the JDK expected by your AGP/Gradle setup, commonly JDK 17 for modern AGP.
compileSdk android/app/build.gradle Dependency requires newer Android API. Increase compileSdk safely; it does not directly increase minimum device support.
minSdk android/app/build.gradle Plugin needs a higher minimum Android version. Increase minSdk only if you accept dropping older devices.

Core Concepts

Terms you must understand.

These are the words you will see repeatedly in Flutter Android build errors.

Gradle

Build automation tool

Gradle runs tasks like compile code, merge resources, run tests, and package APK/AAB.

AGP

Android Gradle Plugin

AGP adds Android-specific behavior to Gradle. Without AGP, Gradle does not know Android app rules.

Plugin DSL

Modern plugin syntax

The plugins {} block is cleaner than old apply from style.

Repository

Download source

google(), mavenCentral(), and gradlePluginPortal() tell Gradle where to download things.

Dependency

External library

Dependencies can be app libraries, test libraries, or transitive libraries coming from other libraries.

Namespace

Android code package

Modern Android projects require namespace inside the android {} block.

Build type

debug / release

Debug is for development. Release is for Play Store or production builds.

Flavor

staging / production

Flavors allow different app names, API URLs, icons, or package IDs from the same codebase.

NDK

Native toolchain

NDK is needed when plugins or your app use native C/C++ libraries.

Command Center

Commands for real debugging.

Use these before changing random Gradle versions.

Check Flutter + Android + Java

This shows Flutter SDK, Android SDK, Java path, and connected devices.

flutter doctor -v

Check project Gradle version

This uses your project Gradle wrapper, not global Gradle.

cd android
./gradlew -v

Build with detailed stacktrace

Use this when Flutter output hides the detailed Gradle reason.

cd android
./gradlew assembleDebug --stacktrace

Inspect dependency tree

Useful for duplicate class and transitive dependency problems.

cd android
./gradlew :app:dependencies

Stop daemon and clean safely

Use after changing Gradle, AGP, Kotlin, or JDK versions.

cd android
./gradlew --stop
cd ..
flutter clean
flutter pub get

Build release APK or AAB

Use APK for testing and AAB for Play Store upload.

flutter build apk --release
flutter build appbundle --release
